# Design Principles

## Purpose
- このプロジェクトのUIは `compact / practical / neutral` を基本方針とする
- LP的な華やかさより、毎日使う実アプリとしての自然さと一覧性を優先する
- github.comやvercel.comのようなイメージ

## Core Principles
### 1. Information First
- コンポーネント単体の見栄えより、画面全体の情報密度と読みやすさを優先する
- 主役はデータと操作導線であり、装飾は補助に留める

### 2. Border Based UI
- 区切りは `shadow` ではなく `border` と背景差で表現する
- 影は原則使わない。ホバーも背景色や border 色の変化で示す

### 3. Compact Layout
- 余白は必要十分に絞る
- セクション間は `24-32px`、要素間は `8-12px`、コンポーネント内は `12-16px` を基準にする
- 可能な限り 8px グリッドに寄せる

### 4. Small Radius
- 基本の角丸は `rounded-md` 以下を使う
- `rounded-xl` 以上は原則避ける
- チップや細かいラベルは `rounded-sm` を優先する

### 5. Quiet Metadata
- タグや補助情報は主張しすぎない
- タイトル、主画像、主要アクションより先に目に入る見た目にしない
- 小さめサイズ、低彩度、薄い背景か枠線中心で扱う

## Component Guidance
### Header / Search
- ヘッダーは操作バーとして簡潔に見せる
- 検索欄やフィルター導線は大きくしすぎず、一覧より目立たせない

### List Items
- 「大きなカード」ではなく「枠付き一覧項目」に寄せる
- 一件あたりの高さを抑え、一覧としての走査性を上げる
- PCでは必要に応じて列数を増やし、横幅あたりの情報量を上げる

### Detail Panels
- 詳細画面も大きなヒーローカードを避け、一覧と同じ密度感に揃える
- 編集UIや履歴UIも過度に目立たせず、実務的なパネルとしてまとめる

## Typography
- タイトルは必要以上に大きくしない
- 太字は使いすぎない
- 補助テキストはコントラストを少し落とし、情報の優先順位を明確にする

## Avoid
- 強い `shadow`
- 大きな角丸
- 広すぎる padding
- 彩度の高いチップ
- いかにも `shadcn/ui` デフォルトに見えるカード感
